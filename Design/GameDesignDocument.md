# 소울 아처 게임 설계서

## 문서 상태

- 상태: `draft`
- 대상: 학교 융합 프로젝트용 Unity 2D 프로토타입
- 원안: `3607 손영하-융프 계획서.hwpx`

## 1. 게임 개요

게임 제목은 **소울 아처(Soul Archer)**다.

제목의 `소울`은 마왕 기사에게 지배당한 몬스터를 처치할 때 해방되는 영혼 조각을 뜻한다. 궁수는 영혼 조각을 모아 상점에서 여정을 돕는 물품과 교환한다. 별도의 소울 게이지나 필살기 시스템은 사용하지 않는다.

플레이어는 우연히 만난 엘리아 공주에게 첫눈에 반한 초보 궁수 레온이다. 마왕 기사 모르칸은 엘리아가 지닌 왕가의 영혼으로 마왕의 성 지하에 봉인된 힘을 깨우기 위해 공주를 납치한다. 레온은 활을 들고 숲, 동굴, 성 외곽을 지나 마왕의 성으로 향한다. 여러 방에서 적을 처치하고 능력을 조합해 모르칸을 쓰러뜨리면 엘리아를 구출할 수 있다.

이동 중에는 공격하지 않으며, 멈춰 있을 때 가장 가까운 적을 자동 조준해 화살을 발사한다. 따라서 전투의 핵심 판단은 공격할 위치와 회피할 시점을 선택하는 것이다.

### 등장인물

| 인물 | 역할 |
|---|---|
| 레온 | 모험 경험은 부족하지만 엘리아를 구하기 위해 활을 든 주인공 |
| 엘리아 | 왕가의 영혼을 지닌 공주이자 빛으로 레온을 돕는 인물 |
| 모르칸 | 왕가의 영혼으로 봉인된 힘을 깨우려는 마왕 기사 |

### 이야기 흐름

```text
공주와의 우연한 만남
→ 공주의 납치
→ 스테이지 1: 숲
→ 숲 보스
→ 스테이지 2: 동굴
→ 동굴 보스
→ 스테이지 3: 마왕의 성
→ 마왕 기사 처치와 공주 구출
```

### 도입 컷신

새 게임을 선택하면 픽셀 일러스트 3장과 장면별 짧은 문장으로 이야기를 전달한다.

| 장면 | 화면 | 문장 초안 |
|---|---|---|
| 1. 만남 | 숲길에서 레온과 엘리아가 우연히 마주친다. | "숲길에서 만난 순간, 레온은 엘리아에게 마음을 빼앗겼다." |
| 2. 납치 | 모르칸이 나타나 엘리아를 붙잡고 어둠 속으로 사라진다. | "모르칸은 왕가의 영혼을 노리고 엘리아를 납치했다." |
| 3. 출발 | 레온이 활을 들고 숲 너머의 성을 바라본다. | "레온은 엘리아를 구하기 위한 모험을 시작했다." |

- 클릭, `Space` 또는 `Enter`로 다음 장면으로 진행한다.
- `Esc`를 길게 누르면 컷신 전체를 건너뛸 수 있다.
- 한 문장은 최대 두 줄로 제한하고 그림을 가리지 않는 하단 영역에 표시한다.
- 별도 캐릭터 이동이나 대화 애니메이션은 만들지 않는다.
- 컷신 3장은 같은 해상도와 제한된 색상 팔레트를 사용한다.

### 엔딩 컷신

마왕 기사를 처치하면 결과 화면 전에 픽셀 일러스트 2장을 표시한다.

| 장면 | 화면 | 문장 초안 |
|---|---|---|
| 1. 구출 | 열린 감옥 앞에서 레온과 엘리아가 다시 만난다. | "마침내 레온은 모르칸을 쓰러뜨리고 엘리아를 구해 냈다." |
| 2. 귀환 | 레온과 엘리아가 아침 햇빛 속에서 왕국으로 돌아간다. | "긴 모험은 끝났고, 두 사람은 함께 왕국으로 돌아갔다." |

- 클릭, `Space` 또는 `Enter`로 다음 장면으로 진행한다.
- 두 장면이 끝나면 승리 결과 화면으로 이동한다.
- 도입 컷신과 같은 해상도, 글자 위치와 색상 팔레트를 사용한다.
- 별도 대화, 이동 연출과 후일담 시스템은 만들지 않는다.

### 공주의 간접 지원

- 공주는 감옥에 갇혀 있지만 왕가의 영혼을 통해 약한 빛을 보낼 수 있다.
- 숲 보스와 동굴 보스를 처치하면 공주의 빛이 나타나 플레이어의 하트 1개를 회복한다.
- 같은 빛이 다음 스테이지로 향하는 출구를 활성화해 공주가 여정에 간접적으로 참여한다.
- 빛은 공주의 형상을 직접 보여 주지 않고 왕가 문양과 밝은 입자 효과로 표현한다.
- 별도 대사와 선택지는 사용하지 않는다.

## 2. 핵심 플레이 루프

1. 다음 방에 입장한다.
2. 전투방이면 문이 잠기고 적이 생성된다.
3. 이동으로 공격을 피하고 정지해 자동 공격한다.
4. 모든 적을 처치하면 방을 클리어한다.
5. 능력 보상 전투라면 무작위 능력 3개 중 하나를 선택한다.
6. 각 스테이지의 보스를 처치하고 다음 지역으로 이동한다.
7. 승리 또는 사망 결과를 기록하고 새 게임을 시작한다.

## 3. MVP 범위

### 포함

- 플레이어 1종: 모험 경험이 부족한 궁수
- 일반 적 6종: 스테이지마다 고유 몬스터 2종
- 스테이지별 보스 3종: 거대 나무 정령, 거대 박쥐, 마왕 기사 모르칸
- 3개 스테이지: 숲, 동굴, 마왕의 성
- 스테이지당 플레이 방 5~6개, 한 판에 총 15~18개 방
- 스테이지마다 독립적으로 생성되는 격자형 연결 방
- 전투방, 보상방, 회복방, 상점방, 이벤트방, 보스방
- 자유 탐색, 이전 방 이동, 방문 방 미니맵
- 자동 조준 및 자동 화살 공격
- 체력, 피격, 사망, 승리
- 스테이지당 2회, 한 판 총 6회의 능력 선택
- 결과 데이터 기록

### 제외

- 멀티플레이
- 모바일 조작
- 온라인 계정과 서버 저장
- 장기 성장 및 장비 수집
- 복수 플레이어 캐릭터와 추가 캠페인
- WebGL과 모바일 빌드

## 4. 조작과 카메라

- 이동: `WASD` 또는 방향키
- 선택: 마우스 클릭
- 일시정지: `Esc`
- 카메라: 플레이어를 따라가는 탑다운 추적 시점
- 공격 조건: 이동 입력이 없고 살아 있는 적이 있을 때
- 전투 중 대시, 구르기와 능동 스킬 버튼은 사용하지 않는다.
- 플레이어는 기본 이동과 위치 선정만으로 적 공격을 피한다.
- `Space`는 도입 컷신 진행에만 사용하며 전투 중 기능은 없다.

### 조작법 화면

- 타이틀 메뉴에 `조작법` 항목을 제공한다.
- 별도 연습방, 허수아비와 전투 중 튜토리얼 문구는 만들지 않는다.
- 조작법은 한 화면 안에 아래 내용을 표시한다.

| 입력 | 기능 |
|---|---|
| `WASD` 또는 방향키 | 이동 및 회피 |
| 이동 입력 해제 | 화면 안에서 가장 가까운 적 자동 공격 |
| 마우스 왼쪽 클릭 | 메뉴와 능력 카드 선택 |
| 숫자키 `1`, `2`, `3` | 왼쪽, 가운데, 오른쪽 능력 카드 선택 |
| `Esc` | 일시정지 또는 이전 화면 |
| `Space` / `Enter` | 컷신 다음 장면 |

- 핵심 규칙인 `이동 중에는 공격하지 않고 멈추면 공격한다`를 가장 크게 표시한다.
- 조작법 화면에서 `Esc` 또는 뒤로 버튼을 누르면 타이틀로 돌아간다.

### 카메라와 화면 밖 전투

- 전투방은 화면보다 크게 만들고 카메라는 플레이어를 부드럽게 따라간다.
- 카메라는 방 외곽을 넘어가지 않아 화면 밖의 빈 공간이 보이지 않게 한다.
- 자동 공격은 현재 카메라 화면 안에 있는 적만 대상으로 선택한다.
- 화면 밖 적도 이동과 공격 AI를 계속 실행하며 공격 조건이 맞으면 자동으로 공격한다.
- 화면 밖 적이 공격을 예고하거나 투사체를 발사하면 해당 방향의 화면 가장자리에 경고 표시를 띄운다.
- 화면 안에서 이미 발사된 적 투사체는 적이 화면 밖으로 나가도 사라지지 않는다.
- 현재 화면에 적이 없으면 가장 가까운 남은 적의 방향을 화면 가장자리 화살표로 표시한다.
- 보스도 화면 밖에서 패턴을 계속할 수 있지만 공격 예고와 방향 경고가 화면 안에서 식별되어야 한다.

## 5. 전투 규칙

### 플레이어 초기값

| 항목 | 초기 제안값 |
|---|---:|
| 최대 체력 | 하트 5개 |
| 이동 속도 | 초당 5 유닛 |
| 공격력 | 10 |
| 공격 간격 | 0.7초 |
| 투사체 속도 | 초당 12 유닛 |
| 피격 무적 | 0.5초 |

수치는 첫 플레이테스트 전 가설이며 확정값이 아니다.

### 체력과 피해

- 플레이어 체력은 하트 5개로 표시하며 반 칸 단위까지 감소하거나 회복할 수 있다.
- 모든 일반 적과 보스의 몸에 닿으면 하트 0.5개의 접촉 피해를 받는다.
- 접촉 피해를 받으면 플레이어가 적 반대 방향으로 짧게 밀려나 즉시 다시 겹치지 않게 한다.
- 일반 몬스터 공격은 패턴 위험도에 따라 하트 0.5개 또는 1개 피해를 준다.
- 보스의 일반 공격은 하트 1개, 강하게 예고된 공격은 최대 1.5개 피해를 준다.
- 한 번 피격되면 0.5초 동안 무적이 되어 같은 공격에 연속으로 피해받지 않는다.
- 피격 무적 중에는 접촉과 다른 공격을 포함한 모든 추가 피해를 받지 않는다.
- 접촉 피해와 예고된 물기, 돌진, 투사체 피해는 서로 다른 판정이며 동시에 맞아도 한 번의 무적 시간만 시작한다.
- 회복으로 최대 하트 수를 넘을 수 없다.

### 자동 공격

- 현재 화면과 공격 가능한 거리 안의 살아 있는 적 중 화살 직선 경로가 열린 가장 가까운 대상을 선택한다.
- 가장 가까운 적이 벽 뒤에 있으면 경로가 열린 다음 적을 찾고, 공격 가능한 적이 없으면 화살을 발사하지 않는다.
- 이동 입력이 끝나면 별도의 조준 대기 없이 첫 화살을 즉시 발사한다.
- 정지 상태를 유지하면 공격 간격마다 같은 방식으로 화살을 반복 발사한다.
- 이동을 시작하면 새로운 공격 발사를 즉시 중단한다.
- 이미 발사된 화살은 계속 이동한다.
- 같은 거리의 적이 여럿이면 먼저 탐색된 적을 선택해 조준 흔들림을 막는다.
- 대상이 사망하거나 사라지면 다음 공격 시 다시 탐색한다.

### 투사체와 지형 충돌

- 기본 화살은 벽과 충돌하면 피해를 주지 않고 사라진다.
- `반사` 능력을 선택한 화살만 선택 횟수만큼 벽에서 튕길 수 있다.
- 반사된 화살은 자동으로 새 적을 추적하지 않고 반사 각도를 따라 직선으로 이동한다.
- 장식 타일과 바닥 효과는 화살을 막지 않는다.
- 일반 적 투사체도 벽에 닿으면 사라지며, 수정탄과 반사 음파처럼 패턴에 명시된 공격만 벽에서 튕긴다.

### 적 역할

아래 AI 역할을 지역별 몬스터가 나누어 사용한다. 같은 역할도 이동 속도, 투사체 형태와 공격 간격을 달리해 지역별 차이를 만든다.

| 역할 | 행동 | 플레이 의도 |
|---|---|---|
| 추적형 | 플레이어를 계속 따라가 접촉 공격 | 이동 경로 압박 |
| 원거리형 | 거리를 유지하며 투사체 발사 | 정지 공격 타이밍 방해 |
| 돌진형 | 예고 후 직선 돌진 | 반응형 회피 요구 |
| 거대 나무 정령 | 뿌리, 씨앗 투사체, 식물 소환 | 기본 전투 숙련 검증 |
| 거대 박쥐 | 비행 돌진, 반사 음파, 암전 | 반응과 위치 선정 검증 |
| 마왕 기사 모르칸 | 방패 압박과 검기를 사용하는 2단계 전투 | 최종 성장 결과 검증 |

### 스테이지별 일반 몬스터

| 스테이지 | 몬스터 A | 몬스터 B |
|---|---|---|
| 숲 | 숲 늑대: 추적형 | 독버섯: 원거리형 |
| 동굴 | 바위 딱정벌레: 돌진형 | 수정 정령: 원거리형 |
| 마왕의 성 | 방패 해골병: 방어형 추적 | 암흑 마법사: 원거리형 |

- 몬스터 6종은 외형과 주 공격이 서로 달라야 한다.
- 이동, 거리 계산, 투사체와 돌진 같은 공통 동작은 같은 역할끼리 공유할 수 있다.
- 한 전투방에는 해당 스테이지의 두 몬스터를 단독 또는 혼합 배치한다.

#### 숲 몬스터

- 숲 늑대: 플레이어를 계속 추적하고 가까워지면 `띠딕` 경고 후 짧게 뛰어들어 물기 공격을 한다.
- 숲 늑대의 역할은 플레이어가 한 자리에 오래 멈춰 공격하지 못하게 압박하는 것이다.
- 독버섯: 느리게 자리를 바꾸며 플레이어 방향으로 큰 포자 탄 1개를 발사한다.
- 포자 탄은 속도가 느리고 크기가 커서 이동 경로를 제한한다.
- 첫 스테이지의 규칙 학습을 단순하게 유지하기 위해 포자 탄은 즉시 피해만 주며 지속 독 상태는 적용하지 않는다.

#### 동굴 몬스터

- 바위 딱정벌레: 잠시 멈춰 직선 경로를 표시하고 `띠딕` 경고 후 벽까지 돌진한다.
- 돌진이 벽에 닿으면 짧게 기절해 안전하게 공격할 기회를 제공한다.
- 수정 정령: 플레이어와 거리를 유지하며 작은 수정탄을 한 발씩 발사한다.
- 수정탄은 벽에 한 번 반사되며 반사 전후에 같은 피해를 준다.
- 바위 딱정벌레와 수정 정령은 거대 박쥐의 돌진과 반사 음파를 낮은 난도로 미리 학습시키는 역할을 한다.

#### 마왕의 성 몬스터

- 방패 해골병: 방패를 든 채 천천히 접근하며 정면에서 오는 화살 피해를 막는다.
- 방패의 좌우와 뒤에서 맞은 화살은 정상 피해를 주며, 공격 직전 방패를 들어 올릴 때도 짧은 약점이 생긴다.
- 자동 조준만으로 공략할 수 있도록 이동으로 각도를 바꾸면 가장 가까운 해골병의 노출된 면을 공격할 수 있다.
- 암흑 마법사: 플레이어와 거리를 유지하며 `띠딕` 경고 후 느린 유도탄을 발사한다.
- 유도탄은 회전 속도와 유지 시간이 제한되어 방향을 크게 바꾸면 피할 수 있고, 벽에 닿으면 사라진다.
- 두 적은 마왕 기사의 방패 측면 공략과 이동 압박을 최종 보스 전에 학습시키는 역할을 한다.

### 숲 보스: 거대 나무 정령

- 역할: 첫 스테이지에서 이동과 정지 공격을 이해했는지 확인하는 보스
- 외형: 방 중앙에 뿌리내린 거대한 고목과 빛나는 핵
- 뿌리 솟구치기: 공격 위치를 바닥에 먼저 표시한 뒤 일렬로 뿌리를 솟게 한다.
- 씨앗 부채꼴: 플레이어 방향으로 피할 틈이 있는 부채꼴 씨앗 투사체를 발사한다.
- 식물 소환: 체력이 일정 이하가 되면 작은 추적형 식물을 최대 2마리 소환한다.
- 약점 시간: 큰 공격이 끝난 뒤 짧게 멈춰 플레이어가 안전하게 연속 사격할 기회를 제공한다.
- 처치 보상: 공주의 빛으로 하트 1개 회복, 동굴로 이동하는 출구 활성화

정확한 체력, 피해량, 공격 간격은 첫 플레이테스트에서 조정한다.

### 동굴 보스: 거대 박쥐

- 역할: 느리고 범위가 넓은 나무 정령과 달리 빠른 방향 전환과 반응형 회피를 시험하는 보스
- 외형: 동굴 천장에 매달렸다가 전투 시작과 함께 날아오르는 거대한 박쥐
- 비행 돌진: 돌진 경로를 짧게 표시한 뒤 방 끝까지 빠르게 날아간다. 벽에 도착하면 잠시 멈춘다.
- 반사 음파: 부채꼴 음파를 발사하며 일부 음파는 벽에 한 번 반사된다.
- 암전: 체력이 일정 이하가 되면 방을 어둡게 하고, 눈과 공격 예고선만 보이는 상태에서 연속 돌진한다.
- 공격 가능 규칙: 비행 중에도 자동 조준과 화살 피해가 적용되어 공격 불가능 시간이 생기지 않는다.
- 약점 시간: 돌진이 끝나 벽에 부딪힌 뒤 짧게 기절한다.
- 처치 보상: 공주의 빛으로 하트 1개 회복, 마왕의 성으로 이동하는 출구 활성화

나무 정령과 패턴이 겹치지 않도록 지면 범위 공격과 몬스터 소환은 사용하지 않는다.

### 최종 보스: 마왕 기사 모르칸

- 역할: 공주를 납치한 장본인이자 플레이어의 이동, 위치 선정과 성장 조합을 모두 시험하는 최종 보스
- 외형: 검은 갑옷, 뿔 달린 투구, 대검과 방패를 든 기사
- 1단계 방패 전진: 플레이어를 향해 천천히 접근하며 정면에서 오는 화살을 막는다. 측면과 후면의 화살은 피해를 준다.
- 1단계 검기: 대검을 휘두르기 전 방향을 예고하고 넓은 초승달 모양 검기를 발사한다.
- 방패 해제: 큰 공격 직후 방패를 내리는 짧은 시간에는 모든 방향에서 피해를 받는다.
- 2단계 갑옷 파괴: 체력이 절반 이하가 되면 방패와 일부 갑옷이 부서지고 이동 속도와 공격 빈도가 증가한다.
- 2단계 연속 참격: 플레이어 주변의 공격 위치를 순서대로 표시한 뒤 세 번의 참격을 가한다.
- 2단계 교차 검기: 서로 교차하는 두 검기를 발사해 이동 경로를 제한한다.
- 사망 결과: 전투가 종료되고 감옥 문이 열리며 공주 구출 장면으로 전환된다.

자동 조준만으로도 공략할 수 있도록 방패는 보스 전체를 무적으로 만들지 않는다. 플레이어가 이동해 각도를 바꾸거나 방패 해제 시간을 이용하면 피해를 줄 수 있다.

## 6. 방 생성

Soul Knight를 참고해 방을 상·하·좌·우 문으로 연결된 격자형 던전으로 생성한다. 플레이어는 열린 문을 통해 자유롭게 탐색하고 이미 클리어한 방으로 돌아갈 수 있다. 숲, 동굴, 마왕의 성은 각각 독립된 맵으로 생성하며, 현재 스테이지의 출구를 통과하면 다음 스테이지로 이동한다.

### 스테이지 구성

| 스테이지 | 환경 | 전투 방향 | 종료 조건 |
|---|---|---|---|
| 1 | 숲 | 기본 적과 이동·정지 공격 학습 | 숲 보스 처치 |
| 2 | 동굴 | 장애물과 강화된 적 조합 | 동굴 보스 처치 |
| 3 | 마왕의 성 | 가장 강한 적 조합과 높은 압박 | 모르칸 처치 |

이전 스테이지에서 획득한 체력, 재화와 능력은 다음 스테이지로 이어진다. 스테이지를 이동하면 이전 맵으로 돌아갈 수 없다.

### 생성 규칙

- 전투가 없는 입장방을 기준으로 플레이 방 5~6개를 연결한다. 입장방은 방 수에 포함하지 않는다.
- 모든 방은 시작방에서 이동 가능한 하나의 연결망에 속해야 한다.
- 일반 방은 상·하·좌·우 중 실제 이웃 방이 있는 방향에만 문을 가진다.
- 각 스테이지의 보스방은 시작방에서 가장 멀리 떨어진 막다른 방 중 하나에 배치한다.
- 보스방 입구는 해당 스테이지의 전투방을 모두 클리어한 뒤 열린다.
- 보스를 처치하면 다음 스테이지 출구가 활성화된다. 마왕 기사를 처치하고 공주를 구출하면 승리한다.
- 전투방 3개, 상점방 1개, 보스방 1개를 반드시 배치한다.
- 여섯 번째 방이 생성되면 보상방 50%, 회복방 30%, 이벤트방 20% 중 하나를 가중치로 선택한다.

### 탐색과 방 상태

- 처음 전투방에 진입하면 문이 잠기고 적이 생성된다.
- 모든 적을 처치하면 방 상태가 `클리어`로 저장되고 문이 열린다.
- 클리어한 방에 다시 들어가도 적은 재생성되지 않는다.
- 보상과 회복처럼 한 번 획득한 요소도 다시 생성되지 않는다.
- 플레이어는 열려 있는 문을 통해 이전 방으로 자유롭게 돌아갈 수 있다.

### 전투방 템플릿

- 숲, 동굴, 마왕의 성에 수제로 설계한 전투방 템플릿을 각각 3개씩 제작한다.
- 한 스테이지의 전투방 3개에는 해당 지역의 세 템플릿을 중복 없이 한 번씩 배정하고 순서만 무작위로 정한다.
- 템플릿은 벽, 장애물, 문, 플레이어 진입 지점과 적 생성 후보 지점을 저장한다.
- 적 종류, 적 수와 실제 생성 후보 지점은 방에 입장할 때 난이도 규칙에 따라 선택한다.
- 플레이어 진입 지점 주변에는 적을 생성하지 않는 안전 영역을 둔다.
- 모든 문 사이에는 플레이어가 이동 가능한 경로가 최소 하나 이상 있어야 한다.
- 장애물이 자동 조준 화살을 과도하게 막지 않도록 방 중앙과 가장자리에 공격 가능한 공간을 함께 둔다.
- 일반 전투방의 크기는 약 1.5~2개 화면 범위로 제한해 적을 찾는 시간이 전투 시간보다 길어지지 않게 한다.
- 보스방, 상점방과 특수방은 전투방 9개와 별도의 고정 템플릿을 사용한다.

| 스테이지 | 템플릿 방향 |
|---|---|
| 숲 | 나무 군집, 작은 연못, 중앙 공터 |
| 동굴 | 바위 기둥, 좁은 통로, 수정 장애물 |
| 마왕의 성 | 석상 기둥, 대칭 벽, 중앙 제단 |

### 미니맵

- 방문한 방은 방 종류와 함께 표시한다.
- 방문한 방에 직접 연결된 미방문 방은 윤곽만 표시한다.
- 현재 방과 연결된 문 방향을 표시한다.
- 보스방은 발견하기 전까지 종류를 숨긴다.

방 종류별 배치 규칙은 다음과 같다.

| 방 종류 | 스테이지당 수 | 기능 |
|---|---:|---|
| 전투방 | 3 | 적 처치 후 재화, 지정된 2개 방은 능력 선택 |
| 상점방 | 1 | 재화로 회복 및 보조 아이템 구매 |
| 보스방 | 1 | 보스 처치 후 다음 스테이지 이동 |
| 보상·회복·이벤트방 | 0~1 | 6개 방이 생성될 때 가중치로 하나 선택 |

스테이지별 방 수를 5개 또는 6개로 정하는 확률은 첫 플레이테스트 전 50%씩 적용한다.

### 이벤트방: 영혼의 샘

- 이벤트방에는 한 번만 사용할 수 있는 영혼의 샘을 배치한다.
- 샘을 선택하면 `50%: 하트 1개 회복 / 50%: 하트 0.5개 피해`를 먼저 표시한다.
- 플레이어가 `사용`을 선택하면 두 결과 중 하나를 동일한 확률로 결정한다.
- 결과가 정해지면 회복은 흰색·금색 빛, 피해는 어두운 보라색 파동으로 구분한다.
- 현재 하트가 0.5개라면 사용 버튼을 비활성화해 이벤트로 즉시 사망하지 않게 한다.
- 사용 여부와 결과는 해당 판이 끝날 때까지 유지하며 방을 다시 방문해도 재사용할 수 없다.
- `지나가기`를 선택하면 샘을 사용하지 않고 방을 나갈 수 있으며 나중에 돌아와 다시 결정할 수 있다.

### 보상방: 영혼 상자

- 보상방에는 영혼 상자 1개를 배치하며 적과 함정은 등장하지 않는다.
- 상자를 클릭하면 영혼 10개를 확정적으로 얻는다.
- 획득한 영혼은 바닥에 떨어지지 않고 즉시 보유 수량에 더한다.
- 상자를 열면 열린 상태를 해당 판이 끝날 때까지 저장하고 재방문해도 다시 보상을 주지 않는다.
- 미니맵에서는 발견 후 보물상자 아이콘으로 표시하고, 획득 후에는 완료 표시를 덧붙인다.

### 회복방: 치유의 석상

- 회복방에는 치유의 석상 1개를 배치하며 적과 함정은 등장하지 않는다.
- 석상을 클릭하면 하트 1.5개를 회복한다.
- 현재 하트가 가득 차 있으면 사용 버튼을 비활성화하고 나중에 다시 방문해 사용할 수 있다.
- 석상을 사용하면 빛이 꺼진 상태를 해당 판이 끝날 때까지 저장하고 재사용할 수 없다.
- 최대 체력을 넘는 회복량은 사라지며 영혼이나 다른 보상으로 바뀌지 않는다.
- 미니맵에서는 발견 후 치유 아이콘으로 표시하고, 사용 후에는 완료 표시를 덧붙인다.

### 능력 보상 배치

- 각 스테이지에서 일반 전투방 1곳과 강화 전투방 1곳이 능력 선택을 제공한다.
- 따라서 스테이지당 2회, 한 판에 총 6회의 능력을 선택한다.
- 강화 전투방은 일반 전투방보다 적 수 또는 적 조합이 어렵고 미니맵에서 별도 아이콘으로 표시한다.
- 능력 보상 전투가 아닌 일반 전투방은 재화 또는 소량 회복을 제공한다.
- 숲과 동굴 보스 처치 후에는 능력 선택 대신 공주의 빛으로 하트 1개를 회복하고 다음 스테이지 출구를 제공한다.
- 마왕 기사 처치 후에는 보상 선택 없이 공주 구출과 결과 화면으로 이동한다.

### 적 웨이브

- 일반 전투방과 일반 능력 보상 전투방은 적이 한 번에 등장하는 1웨이브로 구성한다.
- 강화 전투방만 2웨이브로 구성하며 첫 웨이브를 모두 처치하면 다음 웨이브를 시작한다.
- 두 번째 웨이브 전에 1초의 준비 시간을 주고 적 생성 위치를 바닥 표시로 예고한다.
- 플레이어 바로 옆이나 현재 화면 밖에서는 새 적을 생성하지 않는다.
- 일반 전투는 적 3~5마리, 강화 전투는 웨이브당 2~4마리를 초기 기준으로 사용한다.
- 방 클리어 판정은 마지막 웨이브의 모든 적을 처치한 뒤에만 발생한다.

## 7. 성장 능력

능력 보상 전투를 클리어하면 아직 최대 중첩에 도달하지 않은 능력 중 서로 다른 3개를 무작위로 제시한다. 선택 중 게임 시간은 정지한다.

- 모든 능력은 한 판에서 최대 3회 선택할 수 있다.
- 같은 선택 화면에는 동일한 능력이 두 번 등장하지 않는다.
- 3회 선택한 능력은 이후 후보 목록에서 제외한다.
- 능력 카드는 현재 중첩 수와 다음 선택으로 증가할 효과를 표시한다.
- 체력 회복은 영구 능력 대신 즉시 효과지만 한 판에 최대 3번까지만 후보로 등장한다.

| 능력 | 1회당 효과 | 3회 선택 결과 |
|---|---|---|
| 공격력 증가 | 공격력 +20% | 공격력 +60% |
| 공격 속도 증가 | 공격 간격 -15% | 공격 간격 -45% |
| 관통 | 화살이 적 1명을 추가 관통 | 적 3명 추가 관통 |
| 반사 | 화살이 벽에 1회 추가 반사 | 벽에 3회 반사 |
| 체력 회복 | 하트 1.5개 즉시 회복 | 최대 3회 선택 가능 |
| 이동 속도 증가 | 이동 속도 +10% | 이동 속도 +30% |

공격 간격은 기본값의 50%보다 짧아지지 않도록 하한을 둔다. 모든 수치는 플레이테스트 전 제안값이다.

## 8. 재화와 상점

- 재화의 이름은 `영혼 조각`이며 적 처치 또는 전투방 클리어 시 획득한다.
- 적이 떨어뜨린 영혼 조각에 가까이 가면 자동으로 획득하며 방 클리어 시 남은 조각은 플레이어에게 자동으로 모인다.
- 각 스테이지에 전투가 없는 상점방 1개를 반드시 배치한다.
- 상점은 체력 회복, 보호 부적, 강화 숫돌을 각각 1개씩 판매한다.
- 체력 회복은 하트 1.5개를 즉시 회복한다.
- 보호 부적은 다음에 받을 피해 1회를 무효화한다.
- 강화 숫돌은 현재 스테이지가 끝날 때까지 공격력을 20% 높인다.
- 가격은 모든 스테이지에서 체력 회복 10, 보호 부적 15, 강화 숫돌 20 영혼으로 고정한다.
- 상품은 한 번씩만 구매할 수 있으며 구매 후 재화가 차감된다.
- 영혼 조각, 구매 상품과 임시 효과는 한 판이 끝나면 초기화되고 영구 저장되지 않는다.
- 가격과 재화 획득량은 첫 플레이테스트에서 구매 가능 횟수가 한 판에 2~4회가 되도록 조정한다.

### 영혼 획득 초기값

- 일반 몬스터 1마리를 처치하면 영혼 1개를 얻는다.
- 전투방을 클리어하면 영혼 2개를 추가로 얻는다.
- 숲과 동굴 보스를 처치하면 영혼 5개를 얻는다.
- 모르칸 처치 후에는 상점을 다시 이용할 수 없으므로 영혼을 지급하지 않는다.
- 영혼 획득량과 고정 가격은 첫 플레이테스트 전 가설이며 실제 구매 횟수에 따라 함께 조정한다.

## 9. 성공과 실패

- 승리: 모르칸을 처치하고 엘리아 구출 장면에 도달
- 실패: 플레이어 체력이 0이 됨
- 종료 화면: 결과, 도달 방, 플레이 시간, 선택 능력, 남은 체력 표시
- 사망하면 현재 판은 즉시 종료되며 체크포인트나 부활은 제공하지 않는다.
- 다시 시작하면 첫 스테이지의 입장방부터 시작한다.
- 다시 시작할 때 맵, 능력, 재화, 상점 구매, 임시 효과와 체력이 모두 초기화된다.
- 방 구성과 능력 후보는 새로운 무작위 값으로 생성한다.
- 진행 중인 판은 저장하지 않으며 게임을 종료한 뒤 이어하기 기능을 제공하지 않는다.
- 일시정지 메뉴에서 타이틀로 나가거나 게임을 종료할 때 현재 진행이 사라진다는 확인 창을 표시한다.
- 설정 값과 완료된 판의 전적만 게임 종료 후에도 로컬에 유지한다.

### 난이도

- 새 게임을 선택하면 `쉬움`과 `보통` 중 하나를 선택한다.
- 선택한 난이도는 해당 판이 끝날 때까지 변경할 수 없다.
- 보통은 설계서의 기본 피해량을 그대로 사용한다.
- 쉬움은 모든 적 공격의 피해를 하트 0.5개 줄이되 최소 피해는 하트 0.5개로 유지한다.
- 적 체력, 적 수, 이동 속도, 공격 빈도, 방 생성과 보상 확률은 두 난이도에서 동일하다.
- 난이도 차이를 피해량 하나로 제한해 플레이테스트 결과를 비교하기 쉽게 한다.
- 결과 화면과 플레이테스트 기록에 선택 난이도를 표시한다.

### 목표 플레이 시간

- 한 판의 목표 시간은 시작부터 공주 구출까지 약 20분이다.
- 자유 탐색, 상점 이용과 이전 방 이동 시간을 포함한다.

| 구간 | 목표 시간 |
|---|---:|
| 스테이지 1: 숲 | 약 5분 |
| 스테이지 2: 동굴 | 약 6분 |
| 스테이지 3: 마왕의 성 | 약 8분 |
| 결과 화면 전환 | 약 1분 이내 |

- 일반 전투방은 45~60초 안에 클리어하도록 적 수와 체력을 조정한다.
- 강화 전투방은 60~75초, 보스전은 1~2분을 목표로 한다.
- 첫 플레이에서 길 찾기에 시간이 더 걸리는 것은 허용하되 평균 클리어 시간이 25분을 넘으면 방 또는 적 수를 줄인다.

## 10. 시각 및 음향 방향

- 전체 분위기는 진지하지만 지나치게 잔혹하지 않은 정통 영웅 판타지로 한다.
- 초반의 초보 궁수가 세 스테이지를 거치며 영웅으로 성장하는 인상을 시각 효과와 음악의 강도로 표현한다.
- 저해상도 픽셀아트와 최대 32색의 공통 마스터 팔레트
- 각 스테이지는 공통 팔레트 중 12~16색을 중심으로 사용한다.
- 플레이어와 일반 적의 기본 스프라이트 캔버스는 32×32픽셀을 사용한다.
- 거대 나무 정령, 거대 박쥐와 모르칸의 기본 스프라이트 캔버스는 64×64픽셀로 통일한다.
- 장비와 공격 효과가 캔버스를 벗어날 때는 프레임 크기를 늘릴 수 있지만 캐릭터 몸체 비율은 유지한다.
- 보스 공격 효과는 64×64 영역을 벗어날 수 있지만 보스 몸체의 피격 판정은 보이는 몸체보다 약간 작게 설정한다.
- 스프라이트 확대 시 보간 없이 선명한 픽셀 경계를 유지한다.
- 플레이 화면의 모든 스프라이트는 동일한 픽셀 밀도와 정수 배율을 사용한다.
- 맵 바닥, 벽, 문과 장애물은 16×16픽셀 타일을 기본 단위로 사용한다.
- 32×32 플레이어와 일반 몬스터는 시각적으로 약 2×2타일 크기가 된다.
- 숲, 동굴과 마왕의 성에 각각 별도의 타일 세트를 만들되 충돌 규칙과 문 크기는 공통으로 유지한다.
- 각 지역 타일 세트는 최소한 바닥 변형, 벽 모서리, 문, 막는 장애물과 장식 타일을 포함한다.
- 장식 타일은 이동과 화살을 막지 않으며 충돌 타일과 시각적으로 구분한다.
- 게임 플레이의 내부 기준 해상도는 640×360, 화면 비율은 16:9로 한다.
- 기본 전체화면 출력은 내부 화면을 정수 3배 확대한 1920×1080 FHD로 한다.
- 창 모드는 정수 2배인 1280×720을 기본으로 사용한다.
- 픽셀아트에는 흐림 보간을 사용하지 않으며 화면 확대 시 정수 배율을 유지한다.
- UI는 1920×1080 출력에서 읽을 수 있는 크기로 검증하되 게임 플레이 영역을 과도하게 가리지 않는다.
- 플레이어는 상·하·좌·우와 대각선을 포함한 8방향 모습을 사용한다.
- 좌우 방향은 스프라이트 반전으로 공유해 실제 제작 방향은 위, 위 대각선, 옆, 아래 대각선, 아래의 5세트로 제한한다.
- 이동 중에는 이동 방향을 바라보고, 정지해 공격할 때는 자동 조준 대상과 가장 가까운 8방향으로 몸과 활이 전환된다.
- 플레이어 걷기는 제작 방향별 4프레임으로 구성해 총 20개의 기본 걷기 프레임을 사용한다.
- 플레이어 몸의 별도 공격 애니메이션은 만들지 않고 활이 당겨졌다 돌아오는 짧은 효과로 발사 타이밍을 표현한다.
- 일반 몬스터는 기본적으로 정면 스프라이트만 사용한다. 방향이 중요한 돌진과 원거리 공격은 몸 대신 예고선, 효과와 투사체를 회전해 방향을 전달한다.
- 일반 몬스터는 별도 공격 자세 대신 공격 직전 공통 `띠딕` 경고음과 짧은 점멸 효과를 표시한다.
- 경고 시간은 초기값 0.5초로 두며 빠른 공격도 최소 0.3초의 시각 예고를 보장한다.
- 소리를 끈 상태에서도 공격을 피할 수 있도록 경고음만 사용하는 패턴은 허용하지 않는다.
- 보스는 패턴 판독에 필요한 경우에만 좌우 반전 또는 별도 공격 자세를 추가한다.
- 바닥, 벽, 문은 타일 기반으로 반복 사용
- 숲은 녹색과 밝은 흙색, 동굴은 청록색 수정과 회색 바위, 마왕의 성은 붉은 포인트와 어두운 석재로 지역을 구분한다.
- 플레이어, 적, 적 투사체를 색과 윤곽으로 즉시 구분
- 공격 예고와 피격은 색상만이 아니라 형태 변화와 짧은 효과로 함께 전달
- 필수 효과음: 플레이어 발사, 피격, 적 처치, 문 개방, 선택, 보스 경고
- 음악은 숲의 모험 시작, 동굴의 긴장, 마왕의 성의 결전을 구분하는 스테이지별 1곡씩을 목표로 한다.
- 도입 컷신용 픽셀 일러스트 3장: 만남, 납치, 출발
- 엔딩 컷신용 픽셀 일러스트 2장: 구출, 귀환

### 레온 시각 디자인

- 대표 색상은 선명한 파란 망토와 갈색 가죽 장비다.
- 숲의 녹색과 마왕의 성의 붉은 효과 위에서도 파란 망토가 플레이어 위치를 구분하게 한다.
- 32×32 스프라이트에서 밝은 머리, 파란 망토와 나무 활이 서로 다른 색 덩어리로 읽혀야 한다.
- 가죽 팔 보호대, 작은 화살통과 짧은 장화를 사용해 초보 궁수의 가벼운 장비를 표현한다.
- 중갑, 큰 어깨 장식과 얼굴을 가리는 투구는 사용하지 않는다.
- 피격 시 몸 전체를 밝게 점멸시키고 망토 색만으로 상태를 전달하지 않는다.

### 엘리아 시각 디자인

- 흰 드레스, 금색 왕가 장식과 파란 보석을 대표 요소로 사용한다.
- 작은 금색 왕관과 파란 가슴 보석으로 컷신에서도 공주와 왕가의 영혼을 구분한다.
- 왕가의 빛은 흰색과 금색 입자, 파란 보석 형태의 중심 문양으로 표현한다.
- 레온의 파란 망토와 엘리아의 파란 보석을 같은 계열로 사용해 두 인물의 시각적 연결을 만든다.
- 드레스는 복잡한 무늬보다 큰 흰색 면과 금색 가장자리 위주로 단순화한다.
- 납치와 구출 장면에서 의상 색이 마왕의 성 배경에 묻히지 않도록 밝은 외곽선을 유지한다.

### 모르칸 시각 디자인

- 검은 갑옷, 짙은 붉은 망토와 붉게 빛나는 눈을 대표 요소로 사용한다.
- 뿔 달린 투구, 큰 방패와 대검으로 레온보다 무겁고 위협적인 실루엣을 만든다.
- 1단계는 방패가 몸의 정면을 분명히 가리도록 해 방어 방향을 시각적으로 전달한다.
- 2단계에서는 방패와 갑옷 일부가 부서지고 갑옷 틈에서 붉은 영혼 빛이 새어 나온다.
- 검기와 공격 예고선은 붉은색만 사용하지 않고 밝은 중심선과 형태 변화를 함께 사용한다.
- 엘리아의 흰색·금색 왕가의 빛과 반대되는 어두운 색상 대비를 유지한다.

## 11. UI

- 전투 HUD: 하트 5개, 현재 방, 영혼 조각, 보유 능력
- 일반 적은 처음 피해를 받으면 머리 위에 작은 체력바를 표시한다.
- 일반 적 체력바는 마지막 피해 후 2초가 지나면 숨기고 다시 피해를 받으면 표시 시간을 갱신한다.
- 보스전에서는 화면 상단 중앙에 보스 이름과 큰 체력바를 전투 종료까지 표시한다.
- 체력바는 남은 비율을 길이와 색상으로 함께 표현하며 숫자 체력은 표시하지 않는다.
- 현재 자동 조준 대상으로 선택된 적의 발밑에 작은 금색 조준 고리를 표시한다.
- 조준 고리는 이동 중에도 다음 공격 대상을 보여 주며 대상이 바뀌면 즉시 새 적에게 이동한다.
- 화면 밖, 공격 거리 밖 또는 벽에 가린 적에는 조준 고리를 표시하지 않는다.
- 공격 가능한 적이 없으면 조준 고리를 숨긴다.
- 조준 고리는 적의 붉은 공격 예고와 혼동되지 않도록 금색과 고리 형태를 함께 사용한다.
- 능력 선택 화면은 가로로 나란한 같은 크기의 카드 3개를 표시한다.
- 각 카드는 능력 아이콘, 이름, 현재 중첩 수와 선택 후 한 줄 효과를 항상 표시한다.
- 예시는 `공격력 증가`, `1/3`, `공격력 +20%` 형식으로 통일한다.
- 공격력은 위쪽 화살표가 있는 활, 공격 속도는 시계, 관통은 적을 꿰는 화살, 반사는 벽에서 꺾이는 화살, 회복은 십자 표시 하트, 이동 속도는 장화 아이콘을 사용한다.
- 아이콘의 색상만으로 능력을 구분하지 않고 서로 다른 실루엣과 한글 이름을 함께 사용한다.
- 카드를 클릭하거나 숫자키 `1`, `2`, `3`을 누르면 각각 왼쪽, 가운데, 오른쪽 카드를 선택한다.
- 각 카드 상단에 대응 숫자를 표시한다.
- 첫 선택 입력이 들어오면 즉시 선택을 확정하고 추가 입력을 막은 뒤 나머지 두 카드를 숨긴다.
- 일시정지: 계속하기, 다시 시작, 나가기
- 결과: 승패와 플레이 기록

### 설정 메뉴

- 타이틀 메뉴와 일시정지 메뉴에서 `설정`을 열 수 있다.
- 전체 음량을 0~100 범위의 슬라이더 하나로 조절한다.
- 배경음, 효과음과 `띠딕` 공격 경고음은 모두 전체 음량의 영향을 받는다.
- 창 모드와 전체화면을 전환하는 체크박스를 제공한다.
- 음량과 화면 모드 값은 로컬에 저장해 다음 실행에도 유지한다.
- 배경음·효과음 개별 조절, 해상도 선택과 그래픽 품질 설정은 제공하지 않는다.

## 12. 플레이테스트 데이터

각 판에서 다음 항목을 기록한다.

- 플레이 시간과 승패
- 선택한 난이도
- 사망 또는 클리어한 방 번호
- 선택한 능력 목록
- 영혼의 샘 사용 여부와 결과
- 종료 시 남은 체력
- 방별 받은 피해량
- 주관 평가: 난이도와 재미 각 5점 척도

개인정보 없이 익명 회차 단위로 저장한다. 분석 결과로 적 체력, 적 수, 방 가중치, 능력 효과를 조정한다.

### 전적 화면과 저장

- 승리 또는 사망으로 한 판이 끝나면 결과를 로컬 기기에 자동 저장한다.
- 최근 20판을 유지하며 오래된 기록부터 자동으로 제거한다.
- 온라인 전송, 사용자 이름과 개인정보 수집은 하지 않는다.
- 타이틀 메뉴의 `전적`에서 요약 통계와 개별 판 기록을 확인한다.

| 요약 항목 | 표시 방식 |
|---|---|
| 총 플레이 수 | 숫자 |
| 승리 수와 승률 | 숫자와 백분율 |
| 평균 플레이 시간 | 분:초 |
| 평균 도달 스테이지 | 소수점 한 자리 |
| 가장 많이 선택한 능력 | 능력 이름과 선택 횟수 |

최근 기록 목록에는 플레이 날짜, 난이도, 승패, 플레이 시간, 도달 스테이지·방, 남은 하트와 선택 능력을 표시한다. `기록 초기화`는 확인 창에서 다시 동의해야 실행된다.

## 13. 완료 기준

- 새 게임을 시작해 세 스테이지와 보스 3종을 거쳐 모르칸까지 중단 없이 진행할 수 있다.
- 이동 중 공격하지 않고 정지 시 가장 가까운 적을 공격한다.
- 전투방의 모든 적을 처치하기 전에는 문이 열리지 않는다.
- 방 종류가 설정된 규칙에 따라 생성되고 보스방은 시작방에서 멀리 떨어진 막다른 위치에 배치된다.
- 모든 생성 방이 연결되어 있으며 클리어한 방을 자유롭게 재방문할 수 있다.
- 미니맵이 현재 방, 방문한 방, 발견한 연결 경로를 정확히 표시한다.
- 능력 선택 결과가 이후 전투 수치 또는 투사체 동작에 반영된다.
- 사망과 승리 후 결과를 확인하고 다시 시작할 수 있다.
- 최소 5회의 플레이테스트 데이터를 비교할 수 있다.
- 전적 화면에서 최근 20판과 요약 통계를 확인할 수 있다.

## 14. 확정된 제작 제약

- 최종 제출물은 1920×1080 FHD 전체화면을 기본으로 하는 Windows PC 실행 파일로 제작한다.
- 키보드와 마우스 입력만 지원한다.
- WebGL과 모바일 빌드는 일정과 검증 범위를 줄이기 위해 제외한다.
- 게임 내 컷신, 메뉴, 능력, 상점과 전적 텍스트는 한국어만 지원한다.
- 영어 현지화와 언어 선택 기능은 제작 범위에서 제외한다.
- 한글이 작은 크기에서도 읽히고 배포에 사용할 수 있는 라이선스의 폰트를 사용한다.
